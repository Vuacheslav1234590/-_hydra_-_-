 Hydra - это инструмент для угадывания или для
 взлома пар логина/пароля.
 Программа находится Под лицензией AGPL v3.0.   
 Самая последняя версия всегда доступна по адресу:
                             
https://github.com/vanhauser-thc/thc-hydra

 Пожалуйста, не используйте в военных или
 секретных службах, а также в незаконных целях. 
 (Это желание и ни к чему не обязывает -
 большинство таких людей в любом случае не
 заботятся о законах и этике - и говорят себе,  
 что они одни из хороших.)

 Эти службы не были скомпилированы в: afp
 firebird memcached mongodb mysql5 ncp oracle
 postgres radmin2 sapr3 svn smb2.

 Принцип работы программы:

 hydra [[[-l ЛОГИН или -L ФАЙЛ С ЛОГИНАМИ]
 [-p ПАРОЛЬ или -P ФАЙЛ С ПАРОЛЯМИ]] или
 [-C ФАЙЛ]] [-e nsr] [-o ФАЙЛ ДЛЯ ВЫВОДА]
 [-t ЗАДАЧИ] [-M ФАЙЛ С ЦЕЛЯМИ [-T ЗАДАЧИ]]
 [-w ВРЕМЯ ОЖИДАНИЯ ОТВЕТА] [-W ВРЕМЯ ОЖИДАНИЯ  ОТВЕТА] [-f] [-s ПОРТ]
 [-x МИНИМАЛЬНАЯ ДЛИНА:МАКСИМАЛЬНАЯ:НАБОР СИМВОЛОВ] [-c ВРЕМЯ ОЖИДАНИЯ] [КОМБИНАЦИЯ ОПЦИЙ
 -ISOuvVd46] [-m ОПЦИИ МОДУЛЯ] [cервис:// сервер[:ПОРТ][/ОПЦИИ]]

 ОПЦИИ hydra:

 -R  восстановить предыдущую прерванную
     сессию

 -I  Не обращать внимание на специальный файл
     для восстановления прерванной сессии,
     (не ждать пока 10 секунд пройдет)

 -S  выполнить SSL (т.е с шифрованием)
     соединение

 -s ПОРТ если служба не на порту по умолчанию,      
    то можно задать порт здесь

 -l ЛОГИН или -L ФАЙЛ с ЛОГИНАМИ (именами),
    или загрузить несколько логинов из ФАЙЛА

 -p ПАРОЛЬ или -P ФАЙЛ с паролями для перебора,     
    или загрузить несколько паролей из ФАЙЛА

 -x МИНИМАЛЬНАЯ ДЛИНА:МАКСИМАЛЬНАЯ:НАБОР СИМВОЛОВ
    Генерация паролей для брутфорса, наберите      
    "hydra -x -h" для помощи (все будет на английском,
    предупреждаю)

 -y отключить использоваие символов в брутфорсе (смотрите 
    опцию -x)

 -r дождливый режим для генерации пароля
    (относится к опции -x, смотрите выше)

 -e nsr "n" — пробовать с пустым паролем, "s" — логин в 
    качестве пароля и/или "r" — реверс учётных данных

 -u  зацикливаться на пользователя, а не на
     паролях (эффективно! подразумевается с
     использованием опции -x). То есть
     проверить один пароль для всех логинов.

 -C ФАЙЛ формат где "логин:пароль" разделены
         двоеточиями, вместо опции -L/-P

 -M ФАЙЛ список серверов для атак, каждая
         запись должна быть на новой строке,
         после двоеточия можно задать порт.             
         все это выглядит так:

         сервер:порт  [и с новой строки]
         сервер:порт  [и так далее]

 -o ФАЙЛ выводить найденные пары логин/пароль           
         в ФАЙЛ вместо того, чтобы выводить их
         на экран.

 -b ФОРМАТ указать в каком формате опция -o
           должна выводить данные в файл:
           text (обычный текст), json, jsonv1
           (json 1-вой версии).

 -f / -F выйти, когда пара логин/пароль
         подобрана (-M: -f для хоста, -F
         глобально)

 -t ЗАДАЧИ  количество запущенных параллельно
            ЗАДАЧ или потоков (на хост, по
            умолчанию: 16)

 -w / -W ВРЕМЯ время ожидания ответов
               (32 секунды)/между
               соединениями на поток

 -4 / -6 предпочитать IPv4 (по умолчанию) или
         IPv6 адреса (IP 4-той версии или 6-ой)

 -v, -V, -d  показать подробности, показывать
             логин+пароль для каждой попытки,
             режим отладки

 -O  использовать старые SSL v2 и v3

 -K  не повторять попытки с паролем (обычно
     бывает файл с паролями может содержать
     похожие друг на друга пароли)

 -q  не печатать сообщения об ошибках
     соединения

 -U  подробные сведения об использовании модуля     
     сервер, целями могут быть DNS, IP или
     192.168.0.0/24
     (эта ИЛИ опция -M) "сервис" - служба,
     по которой будет проходить взлом (смотрите
     список поддерживаемых протоколов)

 -m OPT некоторые модули служб поддерживают
    дополнительный ввод (-U для справки по
    модулю, например hydra -U ssh)
                                                                                               
 Для настройки прокси используйте переменные окружения 
 HYDRA_PROXY_HTTP или HYDRA_PROXY и, если необходимо, 
 HYDRA_PROXY_AUTH. Например:

 export HYDRA_PROXY=socks5://127.0.0.1:9150
 (или socks4:// или connect://)

 export HYDRA_PROXY_HTTP=http://proxy:8080     
 export HYDRA_PROXY_AUTH=user:pass

 Примеры использования hydra:

hydra -l Максим -P мои-пароли.txt ftp://192.168.0.1

hydra -L мои-логины.txt -p 1234599 imap://192.168.0.1/PLAIN

hydra -C по-умолчанию.txt -6 pop3s://[2001:db8::1]:143/TLS:DIGEST-MD5                         

hydra -l админ -p пароль ftp://[192.168.0.0/24]/

hydra -L мой-файл-с-логинами.txt -P пароли-для-гидры.txt -M цели-для-гидры.txt ssh


НАСЧЕТ ОПЦИИ:

"-x МИНИМАЛЬНАЯ ДЛИНА:МАКСИМАЛЬНАЯ:НАБОР СИМВОЛОВ"

 Если подбор по словарю не сработал, можно
 применить перебор с автоматической генерацией  символов, на основе заданного набора. Вместо с писка паролей нужно задать опцию -x а ей
 передать строку с параметрами перебора.

 С максимальным и минимальным количеством,  все понятно, они указываются цифрами. В наборе
 символов нужно указать a для всех букв в
 нижнем регистре, A - для букв в верхнем
 регистре и 1 для всех цифр от 0 до 9.
 Дополнительные символы указываются после этой  конструкции как есть. Например:

 4:4:1 - пароль размером четыре символа,
         состоящий только из цифр. А именно все        пароли в диапазоне 0000-9999;

 4:8:1 - пароль от четырех до восьми символов, только из цифр;

 4:5:aA1. - пароль, размером от 4 до 5
            символов, состоит из цифр, букв
            верхнего и нижнего регистра или
            точки.

 Вся команда будет выглядеть вот так:

   hydra -l admin -x 4:4:aA1. ftp://127.0.0.1

 Можно пойти другим путем и указать ip цели и
 порт вручную с помощью опции -s, а затем
 указать модуль:

  hydra -l admin -x 4:4:aA1 -s 21 127.0.0.1 ftp
